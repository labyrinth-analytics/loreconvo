LoreConvo: DB Discovery Fix (fbfdd11) APPROVED

Change: _find_loreconvo_db() and _find_loredocs_db() now check Cowork VM mount paths (/sessions/*/mnt/) before the VM home (~). Fixes data-loss risk: os.path.expanduser("~") in a Cowork VM resolves to the ephemeral VM home, not Debbie's Mac home.

Assessment: Correct. Comment explains the rationale. Brock confirmed as SEC-023 INFO (not a vulnerability). Meg verified 359 tests pass. Architecture is sound.

Minor Observation (Non-blocking, LOW priority)

sorted(glob.glob(...)) gives lexicographic ordering when multiple mounted paths exist. If two Cowork VMs are open simultaneously, the first by sort order wins rather than the most recently active session. Recommend post-mandate: sorted(..., key=os.path.getmtime, reverse=True) to prefer the most recently modified DB. Not a bug in current single-VM usage.

GINA-CI-001: Claude-Mem AI Compression Pattern DEFERRED

Question: Claude-Mem (21.5k stars) uses AI-powered context compression before storage. Should LoreConvo evaluate this pattern?

Decision: DEFERRED -- not for Phase 1. Verbatim storage is correct for LoreConvo. The MemPalace/Claude-Mem AAAK compression was published with an accuracy correction acknowledging underperformance vs. raw mode -- this validates our approach. Adding an LLM pass to the save path introduces latency, cost, and hallucination risk. FTS5 also works best on verbatim text. If storage efficiency becomes a user concern, the summary field (already in schema) is the correct lever, not lossy body compression. Revisit at v1.x if demand warrants.

GINA-CI-002: Hindsight Knowledge Consolidation Pattern DEFERRED

Question: Hindsight v0.4.19 added SOTA-level automatic knowledge consolidation. Should LoreConvo adopt a similar pass?

Decision: DEFERRED -- non-destructive alternative proposed for v0.4. Consolidation is inherently lossy and complex. LoreConvo's atomic-session model is more auditable and predictable. The discoverability benefit can be achieved non-destructively via "session linking" -- auto-tagging related sessions via shared tags or FTS5 overlap overlap, without merging content. File as a v0.4 enhancement. No Phase 1 action needed.

Strategic Assessment

LoreConvo advantages: Zero dependencies, Claude-native one-click install, verbatim accuracy, BSL 1.1 monetizable. MIT means MemPalace can never charge -- they are structurally unable to fund continued development long-term.

LoreConvo gap: FTS5 misses fuzzy/semantic queries. "What did we discuss about performance?" fails if the word "performance" does not appear verbatim.

Short-term recommendation: Messaging should emphasize simplicity, reliability, and Claude-native integration. "Works in one click, never loses your data, no setup" beats ChromaDB complexity for the Cowork user persona.

Medium-term recommendation (post-mandate, v0.4): Prototype sqlite-vec (github.com/asg017/sqlite-vec) as an opt-in semantic search layer. This is a SQLite extension with zero additional Python dependencies -- keeps our dependency story clean while closing the semantic search gap.
